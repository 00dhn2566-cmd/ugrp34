# 기능 계획: 제자리 360° yaw 스핀 (트리거식)

작성: 2026-07-14 (사용자 요구: "이 신호 주면 주변 한 바퀴 spin").
용도: 창문 탐색 스캔 — 비전이 주변을 훑도록 기수를 한 바퀴 돌림. 위치는 유지.

## 설계 결정 (확정)

| # | 항목 | 결정 | 근거 |
|---|---|---|---|
| D1 | 트리거 | **상위 파일(미션 파이프라인)이 `yaw_spin.py`를 실행** → 이 스크립트가 모델 내 불리언 `spin_cmd` 신호(상승 엣지)를 만들어 줌 | 사용자 지정(2026-07-14). 미션 로직은 파이썬 계층 — 시뮬 배치 실행에서는 yaw_spin.py가 `spin_cmd` 타임시리즈를 trajectory.mat에 병합하는 방식, 향후 폐루프(Isaac Sim 연동)에서는 런타임 신호 주입으로 확장 |
| D2 | 랩 처리 | yaw 오차 지점에 `mod(e+π,2π)−π` 삽입 — **선행 필수** | 측정 yaw가 ±180°에서 점프 → 단순 뺄셈 오차는 랩 순간 360° 오차로 폭주. 스핀 없이도 옳은 수정 |
| D3 | 스핀 속도 | 36°/s (한 바퀴 10초), 등속 램프 | yaw 권한의 ~30%만 사용. 가감속 여유 큼 |
| D4 | 방향 | 파라미터 `spin_dir`(+1 CCW / −1 CW), 기본 +1 | |

## 아키텍처

```
[spline_yaw lookup] ──┐
                      ├─> [Yaw Spin Generator] ──> yaw_ref' ──> (랩 보정된) yaw 오차 ──> Control Yaw
[spin_cmd (bool)] ────┘
```

**Yaw Spin Generator** (새 서브시스템, Maneuver Controller 내):
- 엣지 검출: `spin_cmd` 0→1 순간 래치 ON
- 오프셋 생성: Enabled Integrator, 입력 `spin_dir × ω_s`(rad/s), 0→2π 포화
- 완료: 오프셋 ≥ 2π 도달 시 래치 OFF + 오프셋 0으로 리셋 (2π ≡ 0이라 기준 복귀가 무점프)
- 출력: `yaw_ref' = spline_yaw + offset`
- 스핀 중 위치 목표는 건드리지 않음 (위치 루프가 제자리 유지)

## 구현 순서 (단계별 검증 게이트)

**0. 정찰 (시뮬 불필요)**: yaw 오차가 어디서 계산되는지 배선 추적
   - Maneuver Controller / "Altitude and YPR Control" 내 Control Yaw 상류의 Subtract 위치 확인
   - 명령 버스 Element16='yaw'와 측정 Chassis.yaw가 만나는 지점 특정
   - 산출물: 오차 블록 경로 + 현재 랩 처리 유무 판정

**1. 랩 보정 (독립 가치, 먼저 커밋)**
   - 오차 직후에 mod 랩 삽입 (스크립트로 메모리 수정 → 검증 후 굽기)
   - 게이트 T1: ① 기존 호버 회귀 무변화 ② yaw 목표 +170° 고정 명령 시 최단 경로 회전·랩 경계(±180°) 통과 시 무폭주

**2. Spin Generator 구축**
   - 트리거는 시뮬에서 workspace 신호(`spin_cmd` 타임시리즈, 예: t=5s에 1)로 주입
   - 게이트 T2: ① 360°±5° 완주, 소요 10±2s ② 스핀 중 위치 이탈 ≤ 10cm ③ roll/pitch RMS 스펙 유지 ④ 완료 후 기준 yaw 무점프 복귀

**3. 인터페이스 정리 (파이프라인 연결) — `yaw_spin.py` 작성**
   - 위치: `control_seoungjin/yaw_spin.py` (사용자 지정 — control_seoungjin 바로 밑, 상위 미션 파일이 호출하는 진입점)
   - 시그니처(안): `python yaw_spin.py --at 12.0 [--rate 36] [--dir ccw] [--traj trajectory.mat]`
     → trajectory.mat에 `spin_cmd` 타임시리즈(t=12.0에 상승 엣지)를 병합 저장 + 스핀 구간 동안 위치 목표 고정 검증
   - JSON 미션 스키마 항목(예: `{"action":"spin","t":12.0}`)을 이 스크립트 호출로 변환하는 것은 상위 파이프라인 몫 — HANDOFF 세션과 협의
   - 게이트 T3: 이동 궤적 중간 스핀 (정지→스핀→재출발 시퀀스)

**4. 굽기 + 회귀 배터리 전체 재실행 후 커밋**

## 리스크 / 확인사항

- 짐벌락 이력: 스핀 중 roll/pitch 측정 오염 여부 실측 확인 (수평 스핀이라 이론상 안전)
- 위치 루프의 yaw 의존: x/y 오차→roll/pitch 변환이 yaw를 쓰므로, 스핀 중 이 변환이 연속 회전을 따라가는지 T2에서 위치 드리프트로 검증
- 램프 추종 지연(Type 1 SSE): 몇 도 뒤처짐은 스캔 용도상 무해. 필요 시 ω_s 피드포워드(고급기법 단계)로 해소
- ki_yaw는 현재 0(와인드업 회피) — 스핀은 PD로 충분. I 재도입 시 스핀 중 와인드업 재점검 필요

## 선행 의존성

- yaw 게인 확정 (진행 중: 10/0/3 vs 15/0/4 제어량 판정)
- 다음 세션 권장 순서: 정찰(0) → 랩 보정(1) → waypoint 추적 → Spin(2~4)
