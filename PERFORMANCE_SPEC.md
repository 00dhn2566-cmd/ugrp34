# 자세 제어 목표 성능 스펙 (초안 v0.1)

작성: 2026-07-14. PID 정밀 튜닝("깎기") 단계의 채점표.
**모든 항목은 측정 방법이 명시된 것만 스펙으로 인정** — 측정 못 하는 요구는 스펙이 아니다.

## 0. 임무에서의 역산 (왜 이 숫자인가)

최종 임무: 창문(개구부) 통과. 요구 정확도를 창문 여유 ±15cm, 통과 속도 1~2m/s로 가정하면:

```
위치 정확도 ±10cm  ←  위치 루프 대역폭 ~1.5-2 rad/s 필요
                   ←  (캐스케이드 규칙: 내루프는 외루프의 3-5배)
자세 루프 대역폭 ~6-10 rad/s  ←  rise time 0.2-0.3s급
```

즉 roll/pitch 스펙은 "위치 ±10cm를 만들 수 있는 자세 루프"에서 역산된 값이다.

| # | 항목 | 목표 | 현재 실측 | 측정 방법 |

## 2. Yaw (금일 확정 예정 — 참고 기재)

| 항목 | 목표 | 근거 |
|---|---|---|
| rise time | **~1.5s** (0.5~1.5s 허용) | 사용자 지정. 반토크 권한 약함 + 임무상 yaw는 완만한 헤딩 추종 |
| 호버 배회 | ≤ ±3° | roll/pitch 지터가 yaw로 새는 잔여 성분 존재 (R4 개선 시 동반 개선 기대) |
| 외란 잔류 오프셋 | I항으로 소거 (Ti=kp/ki 10~20s) | 결선(diagnose_yaw_verify) 결과로 확정 |
| 포화 여유 | 차등 ≤ limit_yaw×2의 80% | 제어량 채점 기준 |

## 3. 고도 z (정식 스펙 — roll/pitch에 준하는 우선순위)

> 근거(사용자 결정 2026-07-14): z 오버슈트는 곧 바닥/창틀 충돌이며, 창문 통과 정확도는 (y,z) 2차원 문제라
> z는 참고가 아니라 임무 직결 스펙이다. 각도 오버슈트(20% 허용)와 달리 z 오버슈트는 cm 단위로 엄격하게.

| # | 항목 | 목표 | 현재 실측 | 측정 방법 |
|---|---|---|---|---|
| Z0 | 스텝 rise time (1m 고도 변경, 10-90%) | **1~1.5s** (임계감쇠형 — 오버슈트 제약이 지배) | 미측정 | 고도 스텝 응답 테스트 (신규) |
| Z1 | 스텝 오버슈트 (1m 고도 변경 시) | **≤ 5cm** | 미측정 | 동일 |
| Z2 | 호버 고도 유지 | **0.97~1.03m (±3%)** | 0.95~1.00 — 하한 경계선 | verify_hover |
| Z3 | 초반 새그 (이륙 직후 목표 대비 하강) | **≤ 5cm** | ~5cm (0.95까지) — 경계 | 동일, t<2s 구간 |
| Z4 | 외란/기동 중 고도 이탈 | **≤ 10cm** | 토크펄스 중 유지 ✅ | 강건성 배터리 |
| Z5 | 하강 속도 상한 (VRS 회피) | 궤적 제약으로 강제 (path_time) | 미구현 | 궤적 생성기 검사 |

## 3.5 위치 x/y (완화 스펙 — 저속 임무 근거)

> 근거(사용자 결정): 고속 비행이 아니므로 응답 속도는 완화. 단 "느려도 정확히"는 유지.

| 항목 | 목표 | 비고 |
|---|---|---|
| 스텝 rise time | **1~2s 허용** (빠를 필요 없음) | 완화 → roll/pitch rise time 요구도 0.3s에서 완화 가능 (0절 역산 재검토) |
| 위치 드리프트 (10s 호버) | ≤ 5cm | Element0/1=px/py 매핑 확보 — 측정 가능 |
| waypoint 추종 오차 | RMS ≤ 10cm (완만 궤적) | waypoint 테스트에서 확정 |
| 오버슈트 | ≤ 10cm | 창문 접근 시 반대편 프레임 여유 |

## 4. 우선순위와 통과 규칙

0. **축별 우선순위 (사용자 결정)**: roll/pitch = 생존 축(안정성), **z = 임무·안전 축(오버슈트 엄격)**, x/y = 완화 축(느려도 정확히). z 스펙(Z1~Z5)은 roll/pitch에 준하는 우선순위로 취급.
1. **R4/R5 (지터)가 최우선** — 나머지는 대부분 이미 충족이거나 근접. 지터 원인 분리(ki_att=0 / 축간 커플링 / kd 노이즈 증폭 / filtD) 실험부터. Z3(초반 새그)도 경계선이라 동반 확인.
2. 게인 변경 시 **회귀 규칙**: 기존 통과 테스트(호버·자립·토크펄스·CG·yaw)를 전부 재실행해 합격 유지 확인. 하나라도 깨지면 그 게인은 성능이 좋아도 탈락.
3. 스펙 개정은 이 파일에 버전 올려 기록 (v0.1 → v0.2 ...), 근거 없는 숫자 변경 금지.

- Scope 버스 전체 매핑 확보 (2026-07-14): Element0~5 = px/py/pz/pitch/roll/yaw, 10~13 = Prop w, 15/16/21/22/23 = 명령 계열, 17~20 = 모터 전류
